# Case Study 1 – FlyRank ML Internship (SEO Content Refresh)

## Problem

Large websites often contain thousands of webpages, making it difficult for SEO teams to manually identify which pages should be reviewed or refreshed first. Reviewing every page takes time and resources, so the challenge is to prioritize webpages that are most likely to experience declining search performance.

## What I Did

As part of the FlyRank Machine Learning Internship, I am building a machine learning solution to support content refresh decisions. I framed the problem as a binary classification task, explored the starter SEO dataset, identified the unit of analysis, selected an appropriate success metric, and studied how historical SEO signals such as impressions, click-through rate (CTR), average position, content age, and update frequency can be used to predict declining webpages.

I am currently progressing through the internship, completing weekly assignments and improving my understanding of machine learning problem framing, feature selection, model evaluation, and SEO analytics.

## Outcome

This project is currently in progress. So far, I have completed the problem framing stage, explored the dataset, and built an interpretable baseline model using the starter notebooks. Over the coming weeks, I will continue improving the model, engineering new features, validating performance, and building a complete end-to-end machine learning solution for SEO content prioritization.

# Case Study 2 – Employee Management System

## Problem

Managing employee information manually can be time-consuming and error-prone. Organizations need a simple system to store employee records, update information, search employees, and remove records efficiently instead of maintaining everything on paper or spreadsheets.

## What I Did

I developed an Employee Management System to practice object-oriented programming and build a CRUD (Create, Read, Update, Delete) application. The project allows users to add new employees, view employee details, update existing records, search for employees, and delete records through a user-friendly interface. While building the project, I focused on organizing the application into reusable components, handling user input correctly, and designing a clean workflow for managing employee data.

## Outcome

The project strengthened my understanding of software development fundamentals, object-oriented programming, and CRUD operations. It also helped me gain experience in designing applications that solve practical business problems while improving my coding and debugging skills.
